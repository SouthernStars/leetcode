# 62.不同路径
[leetcode 链接](https://leetcode.cn/problems/unique-paths/description/)

### 一、题目描述

一个机器人位于一个 m x n 网格的左上角 （起始点在下图中标记为 “Start” ）。 机器人每次只能向下或者向右移动一步。机器人试图达到网格的右下角（在下图中标记为 “Finish” ）。问总共有多少条不同的路径？

### 二、解题思路

动态规划：
- dp[i][j] 表示从头到达（i,j）w位置的不同路径；初始化大小 m*n, dp[0][0]=1, dp[0][*]=1, dp[*][0]=1
- 状态转移方程：dp[i][j] = dp[i-1][j]+dp[i][j-1], 遍历循序 i: 1=>m-1, j:1=>n-1
- 输出结果：dp[m-1][n-1]

### 三、代码实现

```go
func uniquePaths(m int, n int) int {
    dp := make([][]int, m)
    for i:=0; i<m; i++ {
        one:= make([]int, n)
        dp[i] =  one
    }
    // 初始化
    for i:=0; i<n; i++ {
        dp[0][i] = 1
    }
    for i:=1; i<m; i++ {
        dp[i][0] = 1
    }
    // 状态转移
    for i:=1; i<m; i++ {
        for j:=1; j<n; j++ {
            dp[i][j] = dp[i-1][j] + dp[i][j-1]
        }
    }
    return dp[m-1][n-1]
}
```

# **63.不同路径II**
[leetcode 链接](https://leetcode.cn/problems/unique-paths-ii/description/)

### 一、题目描述

给定一个 m x n 的整数数组 grid。一个机器人初始位于 左上角（即 grid[0][0]）。机器人尝试移动到 右下角（即 grid[m - 1][n - 1]）。机器人每次只能向下或者向右移动一步。网格中的障碍物和空位置分别用 1 和 0 来表示。机器人的移动路径中不能包含 任何 有障碍物的方格。返回机器人能够到达右下角的不同路径数量。

### 二、解题思路

动态规划：
- dp[i][j] 表示从头到达（i，j）位置的路径数量
- 初始化第一行和第一列，如果遇到障碍物则后续位置均为为 0
- 状态转移方程 dp[i][j] = dp[i-1][j] + dp[i][j-1], 如果 (i,j) 是障碍物则为dp[i][j]=0
- 遍历顺序 i:1=>m-1; j:1=>n-1
- 输出结果dp[m-1][n-1]

### 三、代码实现

```go
func uniquePathsWithObstacles(obstacleGrid [][]int) int {
    m, n := len(obstacleGrid), len(obstacleGrid[0])
    dp := make([][]int, m)
    for i:=0; i<m; i++ {
        one := make([]int, n)
        dp[i] = one
    }
    for i:=0;i<n;i++ {
        if obstacleGrid[0][i] == 0 {
            dp[0][i] = 1
        } else {
            break
        }
    }
    for i:=0;i<m;i++ {
        if obstacleGrid[i][0] == 0 {
            dp[i][0] = 1
        } else {
            break
        }
    }
    for i:=1; i<m; i++ {
        for j:=1; j<n; j++ {
            if obstacleGrid[i][j] == 0 {
                dp[i][j] = dp[i-1][j] + dp[i][j-1] 
            } else {
                dp[i][j] = 0
            }   
        }    
    }
    return dp[m-1][n-1]
}
```
