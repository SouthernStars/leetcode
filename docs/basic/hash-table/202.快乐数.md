# 202.快乐数
[leetcode 链接](https://leetcode.cn/problems/happy-number/description/)

### 一、题目描述
编写一个算法来判断一个数 n 是不是快乐数。
「快乐数」 定义为：对于一个正整数，每一次将该数替换为它每个位置上的数字的平方和。然后重复这个过程直到这个数变为 1，也可能是 无限循环但始终变不到 1。如果这个过程结果为 1，那么这个数就是快乐数。如果 n 是快乐数就返回 true；不是，则返回 false。

### 二、解题思路

我们有理由猜测：有以下三种情况：

情况1：最终会得到 1， 例如 7->49->97->130->10->1

情况2：最终会进入一个循环，得不到 1，例如  116->38->73->58->89->145->42->20->4->16->37-->
                                                                                                           ^-------------------------------------------|

情况3：值越来越大，最后接近无穷大。

对于第三种情况，其实是不会存在的，举例对于 3 为数来说，最大的 999 的下一个数是 243，这意味这它要被困在243 以内。9999 的下一位是 324， 9999999999999 的下一位是 1053。
可以看到，4位及以上每一步都会丢失位，最终会在 243 一下进行循环或 得到1。

因此无需考虑第三种情况。

哈希表法：使用 hashtable 记录是否出现循环，如果出现循环返回 false, 否则 true


### 三、代码实现

```c++
bool isHappy(int n) {
    unordered_set<int> seen;
    while (n != 1) {
        if (seen.count(n)) {
            return false;
        }
        seen.insert(n);
        n = next(n);
    }
    return true;
}
int next(int n) {
    int res = 0;
    while (n) {
        res += (n % 10) * (n % 10);
        n /= 10;
    }
    return res;
}
```
